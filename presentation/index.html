As chegadas são estocásticas: λ controla o ritmo

05 | Poisson
λ = 0.3 λ = 0.5 λ = 1.0
  • λ é a taxa média de chegada por segundo.
  • Um processo de Poisson representa chegadas independentes ao longo do tempo.
  • O código sorteia o intervalo até a próxima chegada com delay = -log(u) / lambda.
  • Quanto maior o λ, menor tende a ser o intervalo entre threads chegando à fila.

Uma entidade, uma thread, um ciclo de vida

06 | Modelo concorrente
  • O programa cria uma thread para cada raposa, ovelha e fazendeiro.
  • Cada thread dorme um tempo aleatório, entra na fila e tenta participar de um combo válido.
  • A concorrência real aparece quando várias threads acordam, disputam o mutex e dependem das condições do barco.
sleep_arrival_delay(tipo);
pthread_mutex_lock(&g_mutex);

increment_fila(tipo);
visor_emit_chegou(tipo, id);

while (!embarcou) {
  tentar_liderar_ou_embarcar();
  pthread_cond_wait(&g_cond_embarque, &g_mutex);
}

O estado global é pequeno, mas expressivo

07 | FarmState
fila esquerda r o f Quantos passageiros de cada tipo esperam para embarcar.
barco 3 Ocupação, lado atual, passageiros reservados e já embarcados.
margem direita ✓ Contadores de quem já completou a travessia.

Tudo isso é protegido por g_mutex. A UI não lê memória compartilhada; ela só recebe fotografias desse estado em eventos JSONL.

O predicado separa física de segurança

08 | Combos
int combinacao_valida(int r, int o, int f) {
  int total = r + o + f;
  if (total != 3) return 0;
  if (r == 3 || o == 3 || f == 3) return 1;
  if ((f == 1 || f == 2) && (r > 0 || o > 0)) return 1;
  return 0;
}
  • combinacao_valida() codifica a regra do problema.
  • escolher_combo() percorre uma ordem fixa de combinações válidas.
  • aplicar_combo() reserva os passageiros da fila no barco.

Líder-seguidor organiza a travessia

09 | Sincronização

Antes de partir

Threads chegam, entram na fila e dormem se o predicado ainda não fecha um combo seguro.

pthread_cond_wait(&g_cond_embarque, &g_mutex)
mutex: estado global
cond_embarque
cond_viagem
líder atravessa fora da seção crítica

Depois do combo

O líder emite a travessia; seguidores ficam bloqueados até a chegada segura.

pthread_cond_broadcast(&g_cond_viagem)

A travessia é uma barreira de três passageiros: ninguém desembarca logicamente antes de todos embarcarem e o líder concluir a viagem.

Uma viagem vira uma sequência observável

10 | Fluxo
Chegou 01

Partida 03

Desembarque 04

Retorno 05

O contrato entre C e Pygame é JSONL

11 | IPC
{   "evt": "EMBARQUE", "who": "FAZENDEIRO", "id": 8,   "fila": {"r":0,"o":0,"f":0},   "barco": {"r":1,"o":1,"f":1,"lado":"ESQUERDA"},   "direita": {"r":0,"o":0,"f":0}, "ts": ... }
  • O C escreve uma linha JSON por evento em stdout.
  • Logs humanos ficam em stderr, sem poluir o visualizador.
  • O replay usa ts e dur_ms para preservar o tempo relativo da simulação.
